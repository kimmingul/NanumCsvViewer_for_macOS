# Nanum CSV Viewer v1.8.2

## Highlights

A maintenance release focused on large-file robustness and cleanup. No user-facing feature changes — analytics, charts, and pivots produce identical results.

- Analysis, chart, and pivot scans stream the current view one row at a time and read only the columns each computation needs, instead of materializing the whole filtered view (up to 2,000,000 rows × every column) in memory. This removes a memory-spike / out-of-memory risk on wide files, and a full scan no longer evicts the visible-row cache.
- Temporary Excel/SQLite bridge files and clipboard quick-import files are swept on launch (with an age gate so a document opened at launch is never affected); saved-view bookmarks for deleted files are pruned while bookmarks on unmounted volumes are kept.
- Reviewed by Codex (approach) and Grok (implementation) of the streaming refactor.
- Bumped bundle metadata to `1.8.2(182)`.
